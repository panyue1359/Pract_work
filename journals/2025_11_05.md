- 远端服务器：CentOS 7.5，终端：MobaXterm 25.3，本地：Win 10
- 创建docker容器时添加以下参数：--net=host --privileged=true -e DISPLAY=$DISPLAY -v /tmp/.X11-unix:/tmp/.X11-unix -v $HOME/.Xauthority:/root/.Xauthority:rw [authentication - X11 forwarding of a GUI app running in docker - Stack Overflow](https://stackoverflow.com/questions/44429394/x11-forwarding-of-a-gui-app-running-in-docker)
- x11docker：x11转发Docker图形界面程序的脚本，下载x11docker放置于远端服务器目录，执行`./x11docker 参数 镜像:标签`打开图形界面程序 [mviereck/x11docker: Run GUI applications and desktops in docker and podman containers. Focus on security.](https://github.com/mviereck/x11docker)
	- 参数中使用`-- Docker命令 --`添加Docker命令
	- 执行`./x11docker --network=host ofmonitor:1.0 /ofmonitor/openfoam_monitor`打开CFDmonitor的GUI程序（参考条目“Linux实时残差监控”）会提示“error while loading shared libraries: libz.so.1: failed to map segment from shared object”，无法解决
