- 需求：skills日益增多，需要分门别类管理；混合使用多个CLI，需要灵活增删。现在除Cherry Studio以外的智能体客户端（详见“智能体客户端”）都支持读取和导入主流CLI的技能，已具备统一管理的条件。
- skills-manager：文件夹式管理技能（场景），主流CLI自动同步，切换场景时自动切换CLI下的skills组。另外还支持按项目分发，可以用于管理Cherry Studio和Hello-Halo项目中的skills。不足之处是没有编辑功能 [xingkongliang/skills-manager: A lightweight desktop app to manage, sync, and organize AI agent skills across 15+ coding tools — Cursor, Claude Code, Codex, Copilot, and more.](https://github.com/xingkongliang/skills-manager)
- cc-switch：按CLI管理指令、MCP和skills，支持自定义CLI供应商，此外还可以查看和删除AionUI的会话。不足之处是无法编辑skills或对其分组，也无法检测到skills-manager同步的skills。 [farion1231/cc-switch: A cross-platform desktop All-in-One assistant tool for Claude Code, Codex, OpenCode, openclaw & Gemini CLI.](https://github.com/farion1231/cc-switch)
- Agent Forge：检测指定文件夹中的skills和MCP，编辑功能强大，不具备分发功能，可以作为skills编辑器使用。 [LAIJiangFeng/Agent-Forge: 管理skill和mcp](https://github.com/LAIJiangFeng/Agent-Forge)
- Cherry Studio Skill Manager：Cherry Studio 智能体skills专用管理器，弥补Cherry Studio 只能导入zip以及命名混乱的问题，支持查看、导入、更新三种功能。其中更新功能用于将通过其他途径导入的skills注册到Cherry Studio中，可以和skills-manager配合使用。 [mars2003/cherry-studio-skill-manager: A cross-platform tool for installing and managing Cherry Studio Skills / Cherry Studio Skill 管理工具](https://github.com/mars2003/cherry-studio-skill-manager?tab=readme-ov-file)
