## OpenFOAM并行计算的性能
collapsed:: true
	- ((6920303d-008c-421a-9bae-d436739cb759))
- 关键词：多cpu，跨节点，OpenFOAM parallel performance
- 有论坛用户指出OpenFOAM只是用物理核，因此并行计算分片数应该等于物理核数，而非超线程数。若设置为超线程数，计算速度反而变慢。 [OpenFOAM并行计算中，物理核心数与线程的关系](https://cfd-china.com/topic/2220/openfoam并行计算中-物理核心数与线程的关系/5?_=1762004250143&lang=zh-CN)
- 跨节点并行（2个节点每个10核）的速度反而比使用单个节点（1个节点10核）要慢。论坛用户指出应该是节点之间通信的问题。 [跨节点运行openfoam慢](https://cfd-china.com/topic/2883/跨节点运行openfoam慢)
- 论坛用户指出并行的核数多不一定快，效率的提升和核数的增加不是正比，核数翻倍效率提升不到1.5 [在工作站中测试openfoam并行效率很低，大佬们帮忙看看是什么导致的呢](https://cfd-china.com/topic/7407/在工作站中测试openfoam并行效率很低-大佬们帮忙看看是什么导致的呢/18), [200万网格并行算力测试（OpenFOAM版本）](https://cfd-china.com/topic/3988/200万网格并行算力测试-openfoam版本/144)
- science + computing 的专家分析了OpenFOAM并行计算效率的影响因素，指CPU频率的影响低于50%，单核缓存容量（Cache size per core）更为重要，核数较少时内存带宽为主要瓶颈，核数较多时节点间通信为主要瓶颈，并行计算方法有很大影响。 [Runtime Performance Optimizations for an OpenFOAM Simulation - YouTube](https://www.youtube.com/watch?v=x_6d4fmzHbU)
