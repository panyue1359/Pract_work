## OpenFOAM正则表达式设置边界条件
collapsed:: true
	- ((6920303c-c2f2-43bf-bfae-c0fe22ee85ed))
- 需求：根据边界的命名规律，批量设置边界条件
- 使用正则表达式批量设置，网友Andreas_F给出了示例，`.`表示任意字符，`*`表示重复任意次数，表达式需要用双引号括起来 [OpenFOAM file syntax -- CFD Online Discussion Forums](https://www.cfd-online.com/Forums/openfoam/79553-openfoam-file-syntax.html#_prevue)
  ``` bash
  "motorBike_.*" means all the boundaries starting with "motorBike_"
  (e.g.
  "motorBike_",
  "motorBike_aaa" ,
  "motorBike_bdccccc",
  "motorBike_bdccccc" ,
  "motorBike_.bd",
  "motorBike_engine%56",
  "motorBike_chain%40")
  
  . Matches any single character
  * Matches the preceding element zero or more times
  
  "motorBike_*" means all the boundaries starting with "motorBike" and having " _" zero or more times.
  (e.g.
  "motorBike",
  "motorBike_",
  "motorBike__",
  "motorBike___") 
  ```

- 网友rioV8给出了一个跨文件夹批量删除指定文件名文件的指令`find . -name 文件名 | xargs -I {} rm {}`，AI给出的简化版本为`find . -name 文件名 -delete`，作用是查找当前文件夹下（包括子文件夹）的指定文件并删除 [visual studio code - Deleting multiple identically named files each in separate folder - Stack Overflow](https://stackoverflow.com/questions/70893651/deleting-multiple-identically-named-files-each-in-separate-folder#_prevue)
- AI给出了一些相关的命令
  ``` bash
  # 在chill2-chill28中查找en.yml
  find chill{2..28} -name en.yml
  # 将chill1中的fvSolution和fvSchemes复制到chill2-chill28中
  echo chill{2..28} | xargs -n1 sh -c 'cp chill1/fvSolution chill1/fvSchemes "$0"'
  ```
