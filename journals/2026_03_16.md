- uv是基于rust的python包管理器，在skills中被普遍使用。特色是解析和安装依赖的速度快。
- 和conda的主要区别是“去中心化”，conda的虚拟环境集中存放在主程序目录下，适合跨项目复用；uv的虚拟环境依赖于项目，便于隔离。此外，conda可以处理多种语言的依赖，uv只能处理python。两者是互补关系。 [conda已死，uv当立 - 果冻虾仁的文章 - 知乎](https://zhuanlan.zhihu.com/p/2008680578095002270)
- 作为pip的替代品使用：执行`uv pip install 包名`安装python包，比较方便的是在conda虚拟环境中安装python包时，只要base环境中有uv，则所有虚拟环境都可以直接使用。[conda已死，uv当立 - 果冻虾仁的文章 - 知乎](https://zhuanlan.zhihu.com/p/2008680578095002270)
- 虚拟环境管理：使用cd命令进入项目文件夹，执行`uv venv --python 3.12 --seed 包名`即可创建虚拟环境，执行`source .venv/bin/activate`激活虚拟环境，执行`deactivate`退出虚拟环境，`uv run python main.py`可直接激活虚拟环境并运行脚本。[conda已死，uv当立 - 果冻虾仁的文章 - 知乎](https://zhuanlan.zhihu.com/p/2008680578095002270)
- 项目管理：执行`uv init`将生成项目文件。当有“pyproject.toml”存在时，执行`uv sync`即安装所有依赖，这取代了`pip install -r requirements`。执行`uv add 包名`将添加依赖至“pyproject.toml”文件。[conda已死，uv当立 - 果冻虾仁的文章 - 知乎](https://zhuanlan.zhihu.com/p/2008680578095002270)
- 更换python版本：删除现有的虚拟环境（手动），创建虚拟环境（venv命令），自动安装依赖（sync命令）[conda已死，uv当立 - 果冻虾仁的文章 - 知乎](https://zhuanlan.zhihu.com/p/2008680578095002270)
