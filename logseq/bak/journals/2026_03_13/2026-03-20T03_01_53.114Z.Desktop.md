- 实际上是给CLI做了个界面，组件由“智能体+技能（MCP/Skills）+工作空间”组成，相比聊天机器人，还普遍具有定时任务、控制浏览器、文件预览功能，但是不能让大模型重新回答问题或者生成聊天分支，不满意只能重开。所以，这类客户端的定位不是聊天（如 Cherry Studio、Chatbox等），而是使用大模型执行预先定义好的工作流。

- 使用柏拉图AI提供大模型支持，使用claude-haiku-4-5-20251001模型，安装paper-reading-skills 分析pdf文档 [Max-astro/paper-reading-skills: 一个用于结构化论文精读、批判评审与长期知识沉淀的轻量工作流的 SKILL](https://github.com/Max-astro/paper-reading-skills)
- Cherry Studio：自带创建Agent功能，支持导入skills，但是输出过程中经常中断，或者找不到工作空间里的文件
- AionUi：多CLI管理工具，自带Gemini CLI，感觉大模型的反应速度比其他客户端慢，paper-reading-skills 依赖的 pdf skills 无法安装，预览文件的功能做的不错 [iOfficeAI/AionUi: Free, local, open-source 24/7 Cowork app and OpenClaw for Gemini CLI, Claude Code, Codex, OpenCode, Qwen Code, Goose CLI, Auggie, and more | 🌟 Star if you like it!](https://github.com/iOfficeAI/AionUi)
- Halo：自带Claude Code，反应迅速，执行准确。这个软件的独特之处是“数字人”用于执行循环定时任务，数字人和Skills以及MCP放在一起，所有的工作空间使用默认的系统提示词。这种设计我感觉比较符合这类软件的定位。 [openkursar/hello-halo: Open-source Claude Code GUI — like Claude Cowork. Visual AI assistant with remote access, file management, and built-in AI browser.](https://github.com/openkursar/hello-halo?tab=readme-ov-file)
- openakita：配置简单，反应快，执行准确。还有活人感引擎（主动询问）、多层次记忆和反思系统（自动总结聊天信息作为长期知识库）。定位似乎更偏向AI伴侣。 [openakita/openakita: An open-source AI assistant framework with skills and agent architecture](https://github.com/openakita/openakita)
