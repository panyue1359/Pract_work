- 实际上是给CLI做了个界面，组件由“智能体+技能（MCP/Skills）+工作空间”组成，相比聊天机器人，还普遍具有定时任务、控制浏览器、文件预览功能，但是不能让大模型重新回答问题或者生成聊天分支，不满意只能重开。所以，这类客户端的定位不是聊天（如 Cherry Studio、Chatbox等），而是使用大模型执行预先定义好的工作流。
- 技能管理是我比较关注的问题，Hello-Halo的策略是按工作空间管理，Cherry Studio和AionUi是按角色管理，openakita是默认全局加载。我感觉Agent平台里面定义角色是多余的，因为智能体的主要工作是调用skills，自主发挥的空间很小。
- 另一个问题是服从性，能不能正确调用skills工作，而不自作主张。同样的claude-opus-4-5-20251101模型，Hello-Halo的服从性最好，不知道为什么。
- 我个人的需求主要包括：1. 不间断地输入论文，进行一个汇总和比较分析，提出一些创新点，这可能需要一个记忆持久化的功能。3. 处理数据，绘制数据图、机制图、流程图等等。4. 然后1和3前面的内容给出一个初稿。此外，还可能做一些遗留项目重构的工作。

- AionUi：多CLI管理工具，自带Gemini CLI，更新频率高，预览文件的功能做的不错。问题是只有GeminiCLI支持第三方供应商，反应速度似乎比其他客户端慢，不支持斜杠命令，经常找不到用户导入的技能，只能找到CLI默认位置的技能，需要配合其他工具使用。 [iOfficeAI/AionUi: Free, local, open-source 24/7 Cowork app and OpenClaw for Gemini CLI, Claude Code, Codex, OpenCode, Qwen Code, Goose CLI, Auggie, and more | 🌟 Star if you like it!](https://github.com/iOfficeAI/AionUi)
- Cherry Studio：非常全面的聊天客户端，新增的智能体功能支持自定义角色+多工作区，支持斜杠和@命令，标准.claude文件管理角色技能，一键打开VSCode。问题是导入技能不便，输出偶尔中断，服从性不稳定。
- openakita：配置简单，反应快，执行准确。还有活人感引擎（主动询问）、多层次记忆和反思系统（自动总结聊天信息作为长期知识库），目前问题还比较多。 [openakita/openakita: An open-source AI assistant framework with skills and agent architecture](https://github.com/openakita/openakita)
- bitfun：界面简洁美观，支持多CLI，cowork可以完成复杂任务（如调用drawio绘制流程图），主要问题是不明原因的崩溃，更新速度慢。 [GCWing/BitFun: BitFun is a next-generation AI assistant with built-in Code Agent and Cowork Agent.](https://github.com/GCWing/BitFun)
- Hello-Halo：自带Claude Code，反应迅速，支持斜杠命令。这个软件的独特之处是“数字人”用于执行循环定时任务，数字人和Skills以及MCP放在一起，所有的工作空间使用默认的系统提示词。问题是MCP尚不可用，更新速度慢。 [openkursar/hello-halo: Open-source Claude Code GUI — like Claude Cowork. Visual AI assistant with remote access, file management, and built-in AI browser.](https://github.com/openkursar/hello-halo?tab=readme-ov-file)
